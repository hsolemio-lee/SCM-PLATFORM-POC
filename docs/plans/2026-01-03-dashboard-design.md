# Comprehensive Dashboard Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 전체 솔버 흐름을 한 화면에서 확인하고 실행할 수 있는 종합 대시보드 구현

**Architecture:** 메인+사이드 레이아웃으로 선택된 솔버를 크게 표시하고, 나머지 3개 솔버는 사이드바에 축소형으로 표시. 로그 중심 UI.

**Tech Stack:** React, TypeScript, Tailwind CSS (기존 스택 유지)

---

## 1. 전체 레이아웃

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [Pipeline Header - DP → MP → FP → TP]                  [Run All]      │
├───────────────────────────────────────────────────┬─────────────────────┤
│                                                   │  ┌─────────────────┐│
│  ┌─────────────────────────────────────────────┐  │  │ MP (idle)       ││
│  │  선택된 솔버 (메인)                         │  │  │ 로그 미리보기...││
│  │  - KPI 카드 (축소, 한 줄)                   │  │  └─────────────────┘│
│  │  - 시각화 (축소, ~150px)                    │  │  ┌─────────────────┐│
│  │  - 로그 (크게, ~300px+)                     │  │  │ FP (idle)       ││
│  └─────────────────────────────────────────────┘  │  │ 로그 미리보기...││
│                                                   │  └─────────────────┘│
│                                                   │  ┌─────────────────┐│
│                                                   │  │ TP (idle)       ││
│                                                   │  │ 로그 미리보기...││
│                                                   │  └─────────────────┘│
└───────────────────────────────────────────────────┴─────────────────────┘
```

- 메인 영역: ~75% 너비
- 사이드바: ~25% 너비
- 경로: `/` (홈)

## 2. 사이드바 솔버 카드

```
┌─────────────────────────────┐
│  ● MP (Master Planning)     │  ← 상태 아이콘 + 이름
│  ─────────────────────────  │
│  [INFO] LP iteration 25...  │  ← 최근 로그 2-3줄
│  [INFO] Objective: 987,000  │
│  ─────────────────────────  │
│  [Run]                      │  ← 개별 실행 버튼
└─────────────────────────────┘
```

### 상태별 스타일
- `idle`: 회색 점, 비활성 스타일
- `running`: 시안 점 + 깜빡임 애니메이션, 테두리 강조
- `complete`: 녹색 점

### 인터랙션
- 카드 클릭 → 해당 솔버가 메인으로 전환
- Run 버튼 → 해당 솔버 실행 (이전 Stage complete 시에만 활성화)
- 솔버 실행 시 자동으로 메인으로 전환

## 4. Run All 기능

### 동작 흐름
1. [Run All Pipeline] 클릭
2. DP 실행 → 메인에 DP 표시
3. DP 완료 → MP 자동 실행 → 메인에 MP 표시
4. MP 완료 → FP 자동 실행 → 메인에 FP 표시
5. FP 완료 → TP 자동 실행 → 메인에 TP 표시
6. TP 완료 → 전체 완료

### 상태 관리
- 실행 중 "Run All" 버튼 비활성화
- 각 Stage 완료 시 자동 전환
- 사이드바 상태 실시간 업데이트
